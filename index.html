Cloud-Hosted Static Website on AWS

Project Overview

This project involved deploying my personal portfolio website using a suite of Amazon Web Services (AWS) to build a secure, high-performance, and globally scalable hosting solution. The architecture was designed from the ground up to follow modern cloud infrastructure best practices, moving beyond simple hosting to a professional-grade deployment.

AWS Project Architecture

Key Implementation Steps & Architectural Decisions

The deployment was executed in several key phases:

  • S3 Bucket Provisioning: An Amazon S3 bucket was created to provide robust and durable object storage for the website's static files (HTML, CSS, etc.).
  • Global Content Delivery: An Amazon CloudFront distribution was configured as the Content Delivery Network (CDN). This caches the site at AWS edge locations worldwide, dramatically reducing latency for all users.
  • Enforcing Security & Encryption: HTTPS was implemented by integrating a free SSL/TLS certificate from AWS Certificate Manager (ACM) with the CloudFront distribution. A viewer policy was set to automatically redirect all insecure HTTP requests to HTTPS.
  • Securing the Origin: A crucial security measure was implemented by configuring an AWS IAM Origin Access Control (OAC). This ensures the S3 bucket is kept private from the public internet, allowing access only from the CloudFront distribution. This prevents direct access to the files and protects the origin.

Technologies Used: Amazon S3, Amazon CloudFront, AWS Certificate Manager (ACM), and AWS IAM (Origin Access Control).
